1.1 ndarray 对象

numpy.ndarray 是 NumPy 的核心数据结构，全称 N-dimensional array（N 维数组）。它在内存中以一段连续的字节块存储同质（homogeneous）类型的数据，并通过元数据（形状、步幅、数据类型等）描述如何解读这段内存。这种设计使得 NumPy 的批量运算能够直接调用底层 C/Fortran 代码，避免 Python 循环的性能开销。

理解 ndarray 有三个核心概念需要区分：

概念含义示例
shape（形状）各维度的长度，决定数组的"外观"(3, 4) 表示 3 行 4 列
dtype（数据类型）每个元素的类型，决定内存占用和运算行为float64 每元素 8 字节
strides（步幅）在内存中移动到下一个元素需要跨越的字节数(32, 8)，行间跳 32 字节，列间跳 8 字节

ndarray 支持的常见 dtype 如下：

dtype说明字节典型用途
float64双精度浮点（默认）8科学计算
float32单精度浮点4与 PyTorch 对接
int6464 位整数8索引、计数
int3232 位整数4内存敏感场景
bool布尔值1掩码操作
complex128双精度复数16信号处理
视图（View）与副本（Copy）：ndarray 的切片操作通常返回视图，视图与原数组共享底层内存——修改视图会影响原始数据。这是 NumPy 高效内存使用的关键机制，但也是初学者常见 bug 来源。可用 a.copy() 显式创建独立副本，用 np.shares_memory(a, b) 检查两个数组是否共享内存。

python
a = np.arange(12).reshape(3, 4)
# array([[ 0,  1,  2,  3],

1.5 广播机制（Broadcasting）

广播允许不同形状的数组参与运算，无需手动复制数据。规则：从尾部维度开始比较，维度大小相等或其中一个为 1 时兼容。

python
a = np.ones((3, 4))    # shape (3,4)
b = np.array([1,2,3,4]) # shape (4,)  -> 广播为 (3,4)
a + b                  # 每行都加上 b，结果 shape (3,4)

# 列向量 + 行向量
col = np.array([[1],[2],[3]])  # shape (3,1)
row = np.array([10, 20, 30])   # shape (3,)  -> (1,3)
col + row
# array([[11, 21, 31],
#        [12, 22, 32],
#        [13, 23, 33]])
操作数 A shape操作数 B shape结果 shape是否合法
(3, 4)(4,)(3, 4)✅
(3, 1)(1, 4)(3, 4)✅
(3, 4)(3,)—❌ 尾部不匹配
(5, 3, 4)(3, 4)(5, 3, 4)✅

2.1 Tensor 对象

torch.Tensor 是 PyTorch 的核心数据结构，在概念上与 NumPy 的 ndarray 高度对称，都是同质类型的 N 维数组。但 Tensor 在此基础上额外支持两项关键能力，这正是深度学习框架所必需的：

能力NumPy ndarrayPyTorch Tensor
GPU 计算❌ 仅 CPU✅ 可驻留在 CUDA 设备，调用 GPU 并行算力
自动微分❌ 无✅ 设置 requires_grad=True 后自动构建计算图
内存共享可与同进程 Tensor 共享可与 CPU 上的 ndarray 共享（from_numpy）

Tensor 的内部结构与 ndarray 相同，也由数据指针 + 形状 + 步幅（strides）+ 数据类型组成。理解步幅对于分析 view() 是否合法、转置后内存是否连续等问题至关重要：

python
import torch

t = torch.arange(12).reshape(3, 4)
print(t.shape)    # torch.Size([3, 4])
print(t.stride()) # (4, 1)：行间跨 4 个元素，列间跨 1 个元素

# 转置后步幅改变，内存不再连续
t_T = t.T
print(t_T.is_contiguous())  # False
print(t_T.stride())          # (1, 4)

# contiguous() 创建连续副本，之后 view() 才能正常使用
t_T_cont = t_T.contiguous()
print(t_T_cont.is_contiguous())  # True

PyTorch Tensor 有三种主要的存储位置（device）：

  • cpu默认设备，所有操作在 CPU 上执行
  • cuda:0第一块 NVIDIA GPU，矩阵运算速度大幅提升
  • mpsApple Silicon GPU（M1/M2/M3），PyTorch 2.x 支持
跨设备操作会报错：CPU Tensor 和 CUDA Tensor 之间不能直接进行运算，必须先用 .to(device) 或 .cuda() / .cpu() 将它们统一到同一设备。这是初学者最常遇到的运行时错误之一。

2.3 形状操作

python
t = torch.randn(4, 4)

# view vs reshape
y = t.view(16)       # 要求内存连续；失败时抛出 RuntimeError
z = t.reshape(2, 8)  # 更安全，必要时自动复制；推荐优先用 reshape

t.reshape(-1)         # 压平为 1-D

# 轴操作（对应 NumPy 的 expand_dims / squeeze）
t2 = torch.randn(3)
t2.unsqueeze(0)       # shape (3,) -> (1,3)
t2.unsqueeze(1)       # shape (3,) -> (3,1)
t2.unsqueeze(0).squeeze(0)  # 还原

# 转置
t.T                   # 2-D 转置，等同 t.transpose(0,1)
t.permute(1, 0)       # 多维通用，指定新轴顺序

# 拼接
a = torch.ones(2, 3)
b = torch.zeros(2, 3)
torch.cat([a, b], dim=0)   # shape (4,3)
torch.cat([a, b], dim=1)   # shape (2,6)
torch.stack([a, b], dim=0) # 新增维度，shape (2,2,3)
view vs reshape 关键差异：view() 要求 tensor 在内存中是连续的（t.is_contiguous()），否则抛错；reshape() 在非连续时会自动创建副本。日常开发优先使用 reshape()。

2.5 与 NumPy 的互转

python
import numpy as np
import torch

# NumPy -> Tensor
arr = np.array([1., 2., 3.])
t1 = torch.from_numpy(arr)    # 共享内存！修改 arr 会影响 t1
t2 = torch.tensor(arr)        # 复制一份，独立内存（更安全）

# 验证共享内存
arr[0] = 99
print(t1[0])   # tensor(99.) —— 受影响
print(t2[0])   # tensor(1.)  —— 不受影响

# Tensor -> NumPy（需在 CPU 上）
t = torch.ones(3)
n = t.numpy()           # 共享内存
n2 = t.detach().numpy() # 有梯度时必须先 detach()

# GPU tensor -> NumPy（必须先移到 CPU）
# t_gpu = t.cuda()
# n = t_gpu.cpu().detach().numpy()
方向函数内存共享适用场景
NumPy → Tensortorch.from_numpy()是大数据，避免复制开销
NumPy → Tensortorch.tensor()否安全首选
Tensor → NumPy.numpy()是无梯度、在 CPU 上
Tensor → NumPy.detach().numpy()是有梯度时
Tensor → NumPy.cpu().detach().numpy()否(CPU copy)GPU tensor

2.6 自动微分基础（Autograd）

python
import torch

# requires_grad=True 告诉 PyTorch 追踪对该 tensor 的所有操作
x = torch.tensor([2.0], requires_grad=True)
y = x ** 2 + 3 * x + 1   # 构建计算图：y = x^2 + 3x + 1

# 反向传播，计算 dy/dx
y.backward()

# dy/dx = 2x + 3，在 x=2 时 = 7
print(x.grad)  # tensor([7.])

# torch.no_grad()：推理阶段关闭梯度追踪，节省内存和计算
with torch.no_grad():
    z = x * 2  # 该操作不加入计算图
入门阶段只需记住三点：① 需要优化的参数设置 requires_grad=True；② 每次 backward 前将 .grad 清零（optimizer.zero_grad()）；③ 推理/评估阶段用 torch.no_grad() 包裹，避免不必要的内存消耗。

3.1 Series 与 DataFrame

Pandas 有两种核心数据结构，它们之间存在明确的层级关系：DataFrame 由多个 Series 按列组合而成，每个 Series 都共享同一个行索引（Index）。

Series：带标签的一维数组

pd.Series 本质上是一个带显式标签索引的 1-D ndarray。它在 NumPy 数组的基础上增加了一个 Index 对象，用于替代位置编号，使得数据可以通过有意义的标签访问（如日期、名称、ID）。

组成部分说明
.values底层 NumPy ndarray，存储实际数据
.indexIndex 对象，存储标签，可以是整数、字符串、日期等
.dtype元素数据类型，与 NumPy 一致
.nameSeries 的名称，DataFrame 中对应列名

DataFrame：带标签的二维表格

pd.DataFrame 是 Pandas 最核心的数据结构，可以理解为共享同一行索引的若干 Series 的有序字典。它的每一列是一个 Series，每一行由行索引（index）标识，每一列由列索引（columns）标识。

组成部分说明
.index行标签，默认为 0, 1, 2, ...，可自定义为日期、字符串等
.columns列标签，对应各列的名称
.values底层 NumPy ndarray，shape 为 (行数, 列数)
.dtypes各列的数据类型（Series 形式）
.shape行列数元组，如 (100, 5)
Index 对象的重要性：Pandas 的行/列标签由 Index 对象管理。它不是普通的列表，而是一个不可变的、支持集合运算的有序标签集。两个 DataFrame 进行运算时，Pandas 会自动按标签对齐（align），这在处理缺失数据时尤其关键——标签不匹配的位置会自动填充 NaN，而不会像 NumPy 那样直接按位置运算。

2 随机种子固定（可复现实验）

研究实验的可复现性要求同时固定 NumPy、PyTorch、Python 原生随机数。

python
import random, numpy as np, torch

def set_seed(seed: int = 42):
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    if torch.cuda.is_available():
        torch.cuda.manual_seed_all(seed)
        # 关闭 cuDNN 的随机算法（牺牲少量速度换取确定性）
        torch.backends.cudnn.deterministic = True
        torch.backends.cudnn.benchmark = False

set_seed(42)

import torch, pandas as pd, numpy as np

logits = torch.randn(100, 10)   # 100 个样本，10 类

preds_np = preds.cpu().numpy()             # (100,)

results = pd.DataFrame(probs_np, columns=[f'class_{i}' for i in range(10)])
results['prediction'] = preds_np
results['confidence'] = probs_np.max(axis=1)

print(results.head())
results.to_csv('predictions.csv', index=False)
